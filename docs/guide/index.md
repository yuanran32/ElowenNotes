# 平台使用指南

StudyNote 用来沉淀学习过程中的知识、代码实验、项目经验和面试材料。每篇笔记都应该能回答三个问题：为什么学、核心结论是什么、下次怎么用。

## 目录约定

| 目录 | 用途 |
| --- | --- |
| `/frontend/` | 前端基础、框架、浏览器、工程化、性能优化 |
| `/backend/` | Node 服务、接口设计、数据库、部署与工程实践 |
| `/roadmap/` | 阶段目标、学习任务、里程碑和复盘 |
| `/interview/` | 高频题、项目追问、STAR 口播和模拟面试记录 |
| `/resources/` | 文档、课程、工具、博客和可复用素材 |
| `/templates/` | 笔记模板、复盘模板、面试答案模板 |

## 写作规范

1. 标题使用明确主题，例如 `Promise 并发控制`，避免只写 `学习记录`。
2. 开头写清楚背景和目标，说明这篇笔记解决什么问题。
3. 核心概念配最小代码或流程图，避免只复制定义。
4. 结尾保留“问题清单”和“下次行动”，方便复盘。
5. 面试类内容优先使用第一人称口播，保证能直接练习。

## 推荐流程

| 阶段 | 动作 | 判断标准 |
| --- | --- | --- |
| 收集 | 记录资料链接、问题和关键词 | 能回到资料源 |
| 理解 | 用自己的话重写概念与流程 | 能讲给别人听 |
| 实践 | 编写 Demo、复现实验或改造项目 | 有代码或截图证据 |
| 复盘 | 记录错误原因、替代方案和下次行动 | 能避免重复踩坑 |

## 新建笔记清单

- 选择一个清晰主题。
- 复制 [笔记模板](/templates/note-template)。
- 补齐背景、概念、例子、问题和复盘。
- 在对应栏目侧边栏中补充链接。
- 每周检查一次未完成项。
